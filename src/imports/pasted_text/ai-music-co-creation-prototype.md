Design and generate a functional prototype for a beginner-friendly AI music co-creation web app.

Use Linear as the structural reference, Arc as the emotional and brand-tone reference, and Vercel Design as the refinement and card-detail reference.

Project goal:
This app is designed for complete beginners with no formal music training. The core experience is not prompt-in / music-out. Instead, it should help users create a short music draft step by step with AI support.

Core workflow:
1. The user starts on a welcome screen.
2. The user enters a short melody using a simple note input interface.
3. Each note input should feel interactive and beginner-friendly.
4. After the melody is entered, the user clicks an Analyze button.
5. The system returns 2–3 AI suggestion cards.
6. Each suggestion card should include:
   - style direction
   - preset recommendation
   - rhythm advice
   - a short explanation
7. The user compares the cards and selects one direction.
8. The user then enters a light refinement stage.
9. The user can adjust the result through simple controls.
10. The user views a result draft, saves a version, and compares it with a previous version.

Important design requirements:
- The app must feel like a steering interface, not a black-box generator.
- The user should feel: “I made this with AI,” not “AI made this for me.”
- The design should feel premium, dark, calm, and modern.
- It should combine strong structure with a softer emotional tone.
- Avoid a DAW-like professional interface.
- Avoid a generic SaaS dashboard feel.
- Avoid a piano learning app look.
- Avoid large editorial photo collage sections.
- Prioritize clarity, comparison, and guided decision-making.

Visual direction:
- Use Linear’s structure and modular hierarchy for information architecture.
- Use Arc’s softer, more expressive hero feeling for the entry experience.
- Use Vercel Design’s dark card treatment, spacing, and precision for panels and suggestion cards.
- The product should feel premium and carefully designed, but still easy for beginners to understand.

Required screens:
1. Welcome / Entry
2. Melody Input
3. AI Suggestion Cards
4. Refinement Stage
5. Result / Draft Version
6. Saved Versions / Compare

Required controls:
1. Style / preset comparison cards
2. Light refinement controls
3. Save / undo / compare version controls

Melody Input screen requirements:
- Show a simple clickable note input area
- Make it look interactive and playable
- Include buttons such as Clear, Undo, and Analyze
- The input UI should feel friendly to non-musicians
- This screen should visually lean slightly toward Arc in tone, but still keep a clean product structure

AI Suggestion Cards screen requirements:
- Show 3 cards in a clean and highly legible card-based layout
- Make each card visually distinct but consistent
- Each card should clearly show:
  - style
  - preset
  - rhythm advice
  - explanation
- Include Compare and Select interactions
- This screen should visually lean toward Linear and Vercel Design

Refinement screen requirements:
- Show the selected direction at the top
- Add 3 lightweight controls:
  - simpler ↔ richer
  - softer ↔ brighter
  - more stable ↔ more active
- Add Apply changes and Back buttons
- This screen should feel controlled, elegant, and minimal

Result / Compare requirements:
- Show the chosen direction and current settings
- Show clear Save version and Compare with previous actions
- Include a comparison view between two saved versions
- Keep this section highly structured and easy to scan

Style direction examples:
- Dreamy / Calm
- Warm / Grounded
- Energetic / Playful
- Neutral / Lyrical
- Gentle / Reflective

Preset examples:
- Soft Bell + Pad
- Warm Piano + Soft Strings
- Bright Pluck + Light Synth
- Piano + Soft Pad
- Felt Piano + Ambient Pad

Technical realism:
This prototype should visually support the idea that note input gives immediate feedback, and that AI analysis happens only after the Analyze button is clicked. The result should look like a realistic MVP that could later connect to a lightweight inference backend.

Please generate the interface as a clickable web app prototype with a clear end-to-end user flow.


Visual reference direction:

1. Linear
Use Linear as the main structural reference.
I want the product to have a strong sense of hierarchy, dark modern surfaces, clean feature cards, and a highly organized layout.
Use its clarity, modularity, and comparison-friendly structure.
Do not make the product feel like a project management tool or a B2B dashboard.

2. Arc
Use Arc as the emotional and brand-tone reference.
I want the landing / entry experience to feel softer, more expressive, and less like a conventional utility app.
Use its strong hero feeling, playful but elegant branding energy, and more human emotional tone.
Do not directly copy its large photographic collage or lifestyle editorial layout.

3. Vercel Design
Use Vercel Design as the refinement and card-detail reference.
I want the UI to feel polished, restrained, premium, and carefully aligned.
Use its dark card treatment, spacing, fine borders, and clean typography rhythm.
Do not make the product feel like a design documentation site.

Final combined style:
A premium dark UI with calm but expressive brand tone, strong card-based comparison, precise spacing, and a beginner-friendly interactive flow.
This should feel like a thoughtfully designed AI co-creation product, not a DAW, not a developer tool, and not a generic SaaS dashboard.
